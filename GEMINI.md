# Project Mandates: shcore

このプロジェクトは、外部コマンドに依存せず、POSIX sh の組み込み機能のみで構成された堅牢なライブラリを作成することを目的としています。

## 1. 禁止事項 (Strict Prohibitions)

- **外部ユーティリティの使用禁止**: `[`, `test`, `echo`, `printf`, `grep`, `sed`, `awk`, `cat`, `ls`, `mkdir` などの外部ユーティリティを一切使用しないでください。
  - すべての処理はシェルの組み込み機能（Parameter Expansion, `case`, `set`, `read`, `eval` 等）のみで行う必要があります。
  - どうしても出力が必要な場合は、`printf` ではなく変数への格納や、標準エラー出力へのリダイレクト等を検討してください。
- **Bashisms の禁止**: `[[ ... ]]`, `local`, `declare`, 配列 (`var=(...)`) などの Bash 特有の機能は使用せず、純粋な `sh` (POSIX) 仕様に従ってください。
- **サブシェルの最小化**: `$(...)` や `(...)` によるサブシェルの起動は、パフォーマンスと互換性のために最小限に抑えてください。特にループ内での使用は厳禁です。

## 2. 実装基準 (Coding Standards)

- **変数管理**:
  - 関数内のローカル変数は、`__<関数名>_<変数名>` のようにプレフィックスを付け、名前空間の衝突を避けてください（POSIX sh には `local` がないため）。
  - 関数の終了時には、必ず `unset` を使用して内部変数をクリーニングしてください。
- **エラーハンドリング**:
  - 引数不正などの致命的なエラーには、終了ステータス `64` (EX_USAGE) を使用してください。
  - リードオンリー変数への書き込み試行などには `1` を返してください。
- **文字列操作**:
  - パターン照合や置換には、POSIX 規定のパラメータ展開 (`#`, `##`, `%`, `%%`) と `case` 文を駆使してください。

## 3. 検証 (Validation)

- 外部コマンドへの依存が混入していないか、常に注意を払ってください。
