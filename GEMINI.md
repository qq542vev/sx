# Project Mandates: shcore

このプロジェクトは、外部ユーティリティに一切依存せず、POSIX sh の組み込み機能のみで構成された堅牢な標準ライブラリ `shcore` を構築することを目的としています。

---

## 1. 絶対禁止事項 (Strict Prohibitions)

- **外部ユーティリティの使用禁止**: `[`, `test`, `echo`, `printf`, `grep`, `sed`, `awk`, `cat`, `ls`, `mkdir` 等、あらゆる外部コマンドの使用を禁止します。
  - すべての処理はシェルの組み込み機能（Parameter Expansion, `case`, `set`, `read`, `eval` 等）のみで行う必要があります。
  - 出力が必要な場合は、変数への格納（参照渡し）を基本とし、副作用を最小限に抑えてください。
- **Bashisms / 非標準機能の禁止**: `[[ ... ]]`, `local`, `declare`, 配列 (`var=(...)`) 等は使用せず、純粋な POSIX `sh` 仕様に従ってください。
- **サブシェルの最小化**: `$(...)` や `(...)` によるサブシェルの起動は極限まで抑えてください。特にループ内での使用は厳禁です。

## 2. 命名規約とスコープ管理 (Naming & Scope)

POSIX sh には `local` が存在しないため、命名規則によって変数の衝突を完全に防止します。

- **関数名**:
  - **公開 API**: `sx_<機能名>` (例: `sx_str_eq`)
  - **内部関数**: `__sx_<機能名>` (例: `__sx_str_sub`)
- **内部変数名**:
  - 原則として `__<関数名>_<変数名>` の形式を**必須**とします。
  - **公開 API 内の変数**: 末尾にアンダースコアを**付けない**（例: `sx_func` 内では `__sx_func_var`）。
  - **内部関数内の変数**: 末尾にアンダースコアを**付ける**（例: `__sx_func` 内では `__sx_func_var_`）。
  - ※これにより、公開 API が同名の内部関数を呼び出しても、変数の衝突が起こりません。

## 3. データ構造: SX Array

仮想的な配列を以下の変数セットで表現します：

- `変数名`: 配列シグネチャ (`${SX_SIG_ARR}`) を格納。
- `変数名_len`: 要素数（0 以上の整数）。
- `変数名_n`: 各要素の値（n は 0 から始まる連番）。
- 管理: `sx_var_unset` 等の専用関数を用いて操作することを前提とします。

## 4. 実装標準 (Implementation Standards)

- **ターゲット環境**: `yash -efu -o posix` を基準とし、最も厳格な POSIX 準拠を維持します。
- **インターフェース**:
  - **結果変数**: 実行結果を格納する変数名は、原則として**第1引数**で指定（参照渡し）します。
- **ライフサイクル管理**:
  - **強制クリーンアップ**: 関数内で定義した内部変数は、**早期リターンを含め、`return` 直前に必ず `unset`** してください。
- **エラーハンドリング**:
  - 引数不正（Usage Error）: 終了ステータス `64` (`EX_USAGE`)。
  - 権限/操作拒否（Permission Denied）: 終了ステータス `77` (`EX_NOPERM`)。
- **明示的な正常終了の省略**:
  - 関数の最後において、直前のコマンドの終了ステータスが `0`（正常終了）であることが確実な場合、明示的な `return "${SX_EX_OK}"` や `return 0` は記述せず、シェルのデフォルトの挙動に任せてください。
- **文字列操作**:
  - `sed` や `awk` の代わりに、パラメータ展開 (`#`, `##`, `%`, `%%`) と `case` 文を駆使してください。

## 5. 検証プロトコル (Validation)

- **環境の純粋性**: 自動テスト実行時のsx関数呼びたしは必ず**PATH**を空文字に設定し(例: `PATH= sx_str_eq ...`) と設定し、外部コマンドが一切利用できない状態で動作することを確認してください。
- **静的解析**: `shellcheck -s sh` をパスすることを必須とします。

## 6. コミュニケーションと言語 (Communication & Language)

- **使用言語**: AI とのチャット、コード内のコメント、およびドキュメント（例外を除く）における言語は、原則として**日本語**を使用してください。
